Types of Inheritance

In Java, inheritance is where one class obtains or "inherits" the properties of another class. Generally, the term “child class" refers to the class that is inheriting properties while the term "parent class" refers to the class that has its properties inherited. The keyword "extends" indicates that you are trying to inherit the properties of a parent class

Most of the time we will use only two types of Inheritance. Firstly, there’s single inheritance, and it’s as basic as it sounds. Basically, it’s where one child class inherits one parent class, or, more generally, class A extends B. For example, class mammal extends animal. Within class animal, there are things that all animals do such as breath, eat, sleep, etc... However, class mammal specifically has additional traits such as hasHair, liveBirth (except monotremes but we don’t care about them), etc… Thus, a new declared object mammal m can call on properties such as hair, live birth, or even eat and sleep as those were inherited from class animal. Secondly, there’s multilevel inheritance, where single inheritance is stacked into layers. More generally, class A extends B, class B extends C. For example, class cow extends mammal, where class cow might have properties such as quadruped and hamburgerMaker. Thus, a new object cow c can call on the properties of class cow, but also class mammal and class animal

  • Child Class: The class that extends the features of another class is known as child class, sub class or derived class.
  • Parent Class: The class whose properties and functionalities are used(inherited) by another class is known as parent class, super class or Base class.
